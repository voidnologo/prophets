<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete flash card game at /games/flashcards. Features:
- Start screen with a single Start button
- Photo-identification game: leader photo + 4 name choices
- Wrong choices grey out and disable (process-of-elimination feel)
- Correct answer: green highlight + checkmark icon + text label
- Deliberately wrong first answer causes that card to reappear before the session ends
- "Next" button appears only after the correct answer is found
- Score summary: "X out of 15 on your first try!" (or "Perfect score!")
- "Play Again" always shown; "Replay Missed" only shown when 1+ cards were not first-try correct
- Flash Cards nav link is now an active clickable anchor
  </what-built>
  <how-to-verify>
Test on DESKTOP first, then on a real iOS device (Chrome DevTools emulation is insufficient for production verification — per STATE.md decision).

**ROADMAP Success Criteria — verify each:**

1. Start a session. Confirm the first card shows a leader photo with exactly 4 name options. All 4 should be names you recognize as real leaders (no obviously wrong distractors).

2. Select the CORRECT answer on the first try. Confirm:
   - The button turns green
   - A checkmark icon appears
   - Text appears (e.g., "Yes! That's right.")
   - The text does NOT say "Wrong" or "Nope" anywhere on the page
   - A "Next" button appears
   Confirm this satisfies the colorblind rule: would you know it was correct if you couldn't see color? (Check: icon present? text present?)

3. On the NEXT card, deliberately select a wrong answer first. Confirm:
   - The wrong choice greys out
   - You can continue guessing from remaining choices
   - Eventually select the correct answer — confirm the card was added to the queue (it will reappear later in the session — you don't need to complete the whole session to verify, just note whether the deck count increases from the expected value after a wrong first guess)

4. Complete the full session (or skip to summary). On the summary screen:
   - Confirm score shows "X out of 15 on your first try!" (or "Perfect score!" if all correct)
   - Confirm "Play Again" is present
   - Confirm "Replay Missed" appears if you got any wrong on first try (or is absent if perfect)
   - Click "Replay Missed" if available — confirm a new session starts with only the missed cards

**iOS test (real device, not DevTools):**
- Open the GitHub Pages URL on iPhone/iPad
- Verify the game is playable (buttons tap correctly, photo loads, no layout overflow)
- Verify the Flash Cards nav link is tappable and routes to the game

**Tone check:**
- Scan all visible text during a session — confirm "Wrong" and "Nope" do not appear anywhere
  </how-to-verify>
